## Completion workflow

1. Read the [public assignment page](https://skunkworks-academy.github.io/ls1607/roadmap/owasp-top-10-2025-content-sprint) and issue [#13](https://github.com/skunkworks-academy/ls1607/issues/13).
2. Create a working branch, for example `assignment/owasp-top-10-2025-content-sprint`.
3. Complete one deliverable at a time using the provided templates.
4. Commit each meaningful unit of work with a clear message.
5. Update the evidence register with links, decisions, sources and review notes.
6. Open a pull request to `main` and include `Closes #13` in the pull-request description.
7. Address mentor feedback and obtain approval before merge.

## Quality and safety controls

- Explain each risk accurately and in plain language.
- Separate technical risk, business impact and mitigation.
- Use only authorised, safe and non-destructive examples.
- Do not publish secrets, credentials, personal data or identifying infrastructure details.
- Do not include operational exploit instructions that enable misuse.
- Cite primary sources and distinguish facts from opinion.
- Sanitise screenshots, logs and evidence before publication.
- Follow the repository privacy and validation gates.

## Definition of done

- [ ] All nine working files contain completed, publication-ready content.

- [ ] Evidence links and mentor feedback are captured.
- [ ] Repository validation and build checks pass.
- [ ] Pull request is approved and merged.
- [ ] Issue #13 is closed through the merged pull request.
