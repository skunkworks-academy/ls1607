# Pathway References and Sign-Off

## Reference basis

The roadmap is grounded in Luca’s assessment responses and the following development principles:

- Entry-level cybersecurity capability requires operating-system and networking foundations.
- Practical labs and documented projects provide stronger evidence than passive course completion alone.
- Credential attempts should follow readiness evidence and mentor review.
- Secure AI experimentation requires identity, secrets, permission, data-protection and audit controls.
- Regular feedback and peer exposure address Luca’s identified lack of direction and community access.

### Recommended reference frameworks

| Area | Reference use |
|---|---|
| GitHub Foundations | Repository, collaboration and version-control learning objectives |
| Cisco CCST Cybersecurity | Entry-level cybersecurity and operations pathway |
| Microsoft Security, Compliance and Identity Fundamentals | Identity, compliance and Microsoft security concepts |
| CompTIA Security+ objectives | Broad baseline framework for later credential preparation |
| NIST Cybersecurity Framework concepts | Identify, protect, detect, respond and recover thinking |
| OWASP guidance for AI and LLM applications | Prompt injection, tool abuse, data leakage and insecure output risks |

:::info Currency check
Certification codes, exam availability and programme requirements can change. The training team must verify the current official pathway before scheduling any exam or paid learning product.
:::
